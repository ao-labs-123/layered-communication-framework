# 1. Purpose
このデータセットはSNSコメントにおける「**態度の向き先(subject)**」と「**態度の種類(stance)**」を分析するために作成された。

# 2. Layer Annotation
## Definition

A:
コミュニケーション上の認識ズレや参照ズレを扱う発話

B:
対象に対する評価、感情、解釈を含む発話

C:
判断、要求、提案、責任帰属を含む発話

## Layer Selection Rule

最も支配的なLayerを選択する。

A/B/Cの複数付与は行わない。

曖昧な場合は以下を優先する。

C > B > A

アノテーションでは単一Layerを付与する。
解析システムでは各Layerの割合を別途算出する。

# 3. Subject Annotation
## 3.1 基本原則
subjectは登場する名詞ではなく、態度の向き先を選ぶ。

発話内に複数対象が存在する場合、
発話者が最終的に最も強く向けている対象を選択する。
背景説明よりも結論・主張を優先する。
   
## 3.2 Subject Boundary Cases
  - 国 vs 政府 vs 自治体
   
例：
「東京に出ないと国は動かない」

→ 国

⸻

- 熊 vs 自治体

例：
「熊を駆除してほしい」

→ 自治体

理由：
要求の向き先が自治体だから

⸻

- 熊 vs 被害者

例：
「亡くなられた方が気の毒」

→ 被害者

# 4. Stance Annotation
## 4.1 基本原則
   stanceは発話の中心的態度を選ぶ。
   
## 4.2 Stance Boundary Cases
- 批判 vs 皮肉

例：
「熊かわいそうと言ってた人はどう思うんだろう」

→ 皮肉

⸻

- 批判 vs 分析

例：
「暖冬で冬眠できなかったから増えた」

→ 分析

⸻

- 不安 vs 共感

例：
「本当に怖い」

→ 不安

例：
「気の毒でならない」

→ 共感
   
# 5. Multi-label Policy

## Layer

Layerは原則単一ラベル。
最も支配的な層を選択する。

## Subject

Subjectは原則単一ラベル。
発話の態度が最も向いている対象を選択する。

## Stance

Stanceのみ複数ラベルを許可する。

## 許可例

- 批判/要求
- 共感/賞賛
- 不安/要求
- 皮肉/批判

## 非推奨

- 賞賛/批判
- 共感/皮肉
